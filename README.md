# PicPack

PicPack は、指定したフォルダ直下の画像ファイルを作成日時順に並べ、指定枚数ごとのまとまりを固定数のフォルダへラウンドロビン方式でコピーまたは移動する Windows デスクトップアプリです。

たとえば「1フォルダあたりの枚数」が `3`、「使用するフォルダ数」が `7` の場合、`001`, `002`, `003`, `004`, `005`, `006`, `007`, `001`, `002`... の順で振り分けます。

## できること

- `.jpg`, `.jpeg`, `.png`, `.webp`, `.bmp` の画像を対象にします。
- 作成日時の古い順に並べ、同じ作成日時ならファイル名順に並べます。
- 1フォルダあたりの枚数を指定できます。
- 使用するフォルダ数を指定できます。
- 最後のフォルダまで使ったら、次のまとまりは `001` に戻ります。
- コピーまたは移動を選べます。
- 出力先に同名ファイルがある場合は `_001`, `_002` のような連番を付け、既存ファイルを上書きしません。
- 処理中の進捗、ログ、最後のエラー一覧を表示します。
- 実行中のキャンセルに対応しています。

## 使い方

1. `入力フォルダを選択` で画像が入っているフォルダを選びます。
2. `出力フォルダを選択` で振り分け先を選びます。
3. `1フォルダあたりの枚数` を入力します。
4. `使用するフォルダ数` を入力します。
5. `コピーする` または `移動する` を選びます。
6. 実行前確認を確認してから `実行` します。

## 注意事項

- 移動モードは元画像の場所が変わるため、最初はコピーモードで確認してください。
- 作成日時はファイルコピーやダウンロード時に変わることがあります。
- 撮影日時順ではなく、ファイルの作成日時順です。
- 初期実装では入力フォルダ直下のみを対象にし、サブフォルダは検索しません。
- キャンセルした場合、未処理のファイルは処理しませんが、すでにコピーまたは移動済みのファイルは戻しません。

## 開発環境

- .NET 8
- C#
- WPF
- Windows

## ビルド方法

```powershell
dotnet restore
dotnet build PicPack.sln -c Release
```

## テスト方法

```powershell
dotnet test
```
